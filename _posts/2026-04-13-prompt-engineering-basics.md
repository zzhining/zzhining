## 프롬프트 엔지니어링이란?

AI 언어 모델에게 원하는 결과를 얻기 위해 **입력 텍스트(프롬프트)를 설계하는 기술**입니다. 같은 모델이라도 프롬프트 품질에 따라 결과물의 차이가 매우 크게 납니다.

---

## 핵심 기법 5가지

### 1. 역할 부여 (Role Assignment)

모델에게 특정 역할을 부여하면 해당 관점에서 응답합니다.

```
❌ 나쁜 예시:
"파이썬 코드 설명해줘"

✅ 좋은 예시:
"당신은 10년 경력의 파이썬 시니어 개발자입니다. 
다음 코드를 주니어 개발자가 이해할 수 있도록 
단계별로 설명해주세요."
```

### 2. Zero-shot vs Few-shot

**Zero-shot**: 예시 없이 바로 지시

**Few-shot**: 예시를 제공하여 패턴 학습

```
Few-shot 예시:
입력: "나는 밥을 먹었다" → 출력: "I ate rice"
입력: "하늘이 맑다" → 출력: "The sky is clear"
입력: "오늘 날씨가 좋다" → 출력: ?
```

### 3. Chain-of-Thought (CoT)

복잡한 문제를 단계별로 풀도록 유도합니다.

```
"다음 문제를 단계별로 생각하면서 풀어주세요.
각 단계를 명확히 설명하고, 최종 답을 도출하세요."
```

### 4. 구체적인 출력 형식 지정

```
"다음 정보를 JSON 형식으로 정리해주세요:
{
  "title": "제목",
  "summary": "3줄 요약",
  "key_points": ["포인트1", "포인트2", "포인트3"]
}"
```

### 5. 제약 조건 명시

```
"다음 규칙을 반드시 지켜주세요:
- 500자 이내로 작성
- 전문 용어 사용 금지
- 초등학생도 이해할 수 있는 수준
- 예시 2개 이상 포함"
```

---

## 프롬프트 작성 체크리스트

- [ ] 역할과 컨텍스트를 명확히 설정했나요?
- [ ] 원하는 출력 형식을 지정했나요?
- [ ] 제약 조건을 명시했나요?
- [ ] 예시를 제공했나요? (Few-shot)
- [ ] 단계별 사고를 요청했나요? (CoT)

---

## 실무 팁

> **이터레이션이 핵심입니다.** 첫 번째 프롬프트가 완벽할 수 없습니다. 결과를 보고 계속 개선해나가는 것이 프롬프트 엔지니어링의 본질입니다.

좋은 프롬프트 = **명확한 목표** + **충분한 컨텍스트** + **구체적인 출력 요구사항**
